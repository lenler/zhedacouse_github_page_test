# GitHub Pages 更新操作方法

## 当前部署信息

- 仓库名：`zhedacouse`
- GitHub 用户名：`hyyyy051028`
- 站点地址：`https://hyyyy051028.github.io/zhedacouse/`

## 首次部署

1. 确认代码已经推送到 GitHub 主分支。

```bash
git add .
git commit -m "chore: prepare github pages deploy"
git push origin main
```

2. 在项目根目录执行部署命令。

```bash
npm run deploy
```

3. 打开 GitHub 仓库设置。
4. 进入 `Settings` -> `Pages`。
5. 在 `Source` 中选择 `Deploy from a branch`。
6. 分支选择 `gh-pages`，目录选择 `/(root)`。
7. 保存后等待 1-3 分钟。

## 后续更新

代码改完后，按下面步骤更新线上页面：

```bash
git add .
git commit -m "feat: 你的更新说明"
git push origin main
npm run deploy
```

部署完成后，等待约 1 分钟，刷新站点即可看到最新内容。

## 常见问题

### 页面 404

- 检查 GitHub Pages 是否已经切到 `gh-pages` 分支
- 检查访问地址是否为 `https://hyyyy051028.github.io/zhedacouse/`
- 如果是访问子路由时报错，先重新执行一次 `npm run deploy`

### 图片不显示

- 新增图片时，优先放到 `public` 目录
- 代码里不要手写以站点根目录开头的绝对路径，保持走现有的 `withBasePath()`

### 页面改了但线上没更新

- 先确认本地执行过 `npm run deploy`
- 再确认部署命令执行过程中没有报错
- GitHub Pages 可能有 1-3 分钟缓存，稍等后强刷页面再看
