# 网站性能优化文档

## 1. 目标

- 降低首页首屏加载压力，减少大图并发请求。
- 优化非首屏图片加载策略，避免阻塞关键渲染路径。
- 通过按需分包减少首页初始 JavaScript 体积。

## 2. 本次已落地优化

### 2.1 Banner 轮播图加载优化

- 问题：轮播图使用背景图，默认可能触发多张大图并发加载。
- 方案：仅为“当前页 + 前后相邻页”注入背景图 URL，其余页先使用渐变背景占位。
- 收益：降低首屏阶段图片并发下载量，减少带宽峰值与首屏竞争资源。

### 2.2 图片懒加载与异步解码

- Stories 模块从背景图改为 `<img>` 渲染，统一启用：
  - `loading="lazy"`
  - `decoding="async"`
- Intro 模块图片补充 `loading="lazy"` 和 `decoding="async"`。
- Footer 模块 Logo 与二维码图片补充 `loading="lazy"` 和 `decoding="async"`。
- 收益：非首屏图片延后加载，减少初始下载与主线程解码压力。

### 2.3 日历模块按需分包

- 问题：`EventsSection` 作为首页组件会被打进主包，增加首页初始 JS 体积。
- 方案：
  - 将 `EventsSection` 改为 `lazy import`。
  - 仅当 URL 参数触发日历显示时再加载模块。
- 收益：首页默认访问路径下减少无关业务代码的下载与执行。
